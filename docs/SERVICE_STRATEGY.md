# Service Strategy

## 서비스 재정의

이 프로젝트는 더 이상 `서울 구 리포트 대시보드`가 아니다.
핵심 단위는 `구`가 아니라 `검토 중인 매물`이다.

서비스 질문은 다음처럼 바뀐다.

- 어떤 구가 위험한가?
- 내가 지금 검토 중인 이 매물을 사도 되는가?
- 지금 사지 않는다면, 어떤 구를 먼저 비교해야 하는가?

## 핵심 가치

### 1. 매입 보류 신호를 먼저 보여준다

장점보다 반대 근거를 먼저 보여준다.
서비스의 목적은 좋은 매물을 추천하는 것이 아니라, 나쁜 매입을 더 빨리 걸러내는 것이다.

### 2. 설명 가능한 경고를 제공한다

모든 점수는 가격 부담, 거래 유동성, 가격 변동성, 상권 과밀 중 하나와 연결되어야 한다.
숫자 자체보다 `왜 위험한가`가 더 중요하다.

### 3. 대체 행동으로 이어진다

경고는 끝이 아니라 다음 행동으로 이어져야 한다.
예를 들어:

- 현장 답사에서 무엇을 볼지
- 어떤 대체 구를 먼저 비교할지
- 어떤 매물은 바로 보류해야 할지

## 서비스 구조

### 1. 내 매물 검토

- 검토 중인 매물을 입력한다.
- 면적, 희망 매입가, 보유 기간, 우선순위를 넣는다.
- 저장 가능한 레드팀 메모를 생성한다.

### 2. 3분 진단

- 구 단위 빠른 필터링
- 매물 등록 전 1차 판정
- 보류 / 강한 비교 필요 / 보수 검토 / 추가 검토 가능

### 3. 후보 비교

- 2~3개 구를 같은 프레임에서 비교
- 더 안전한 대체 구 탐색
- 가격선과 리스크 축의 차이 확인

### 4. 구별 리포트

- 구 단위 메모
- 최근 12개월 추세
- 리스크 유형
- 체크리스트

## 운영 가설

### 가설 1

사용자는 좋은 매물 추천보다 `사지 말아야 할 이유`를 더 빠르게 이해한다.

### 가설 2

같은 예산 내 대체 후보를 함께 제시하면 경고가 실제 비교 행동으로 이어진다.

### 가설 3

매물 검토 결과를 저장할 수 있어야 단발성 대시보드가 아니라 반복 사용하는 서비스가 된다.

## 제품 KPI

- 첫 진단 완료율
- 저장된 매물 검토 수
- 진단 후 비교 페이지 이동률
- 대체 후보 클릭률
- 현장 체크리스트 확인률

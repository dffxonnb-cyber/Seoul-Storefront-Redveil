# Validation Strategy

## 목적

이 프로젝트의 점수는 추천 점수가 아니라 `매입 반대 근거`를 만드는 점수다.
따라서 검증도 단순 정확도보다 아래 질문에 답해야 한다.

- 왜 이 구가 위험한가?
- 다른 구와 비교했을 때 더 위험한 이유가 명확한가?
- 실제 의사결정에서 다음 행동으로 이어질 수 있는가?

## 검증 프레임

### 1. 구조 검증

리스크 축이 의미 있게 분리되는가?

- 가격 부담
- 거래 유동성
- 가격 변동성
- 상권 과밀

같은 총점이라도 다른 리스크 유형으로 분류되는지 확인한다.

### 2. 사례 검증

상위 위험 구와 대체 후보를 짝지어 케이스 스터디로 검증한다.

현재 기준:

- 서초구
- 강남구
- 중구

향후:

- 10개 구 이상으로 확대

### 3. 신뢰도 검증

최근 거래 표본이 적은 구는 결과를 강하게 주장하지 않는다.

- high sample
- medium sample
- low sample

저표본 구는 결과 화면과 메모에서 별도 경고를 붙인다.

### 4. 서비스 검증

사용자가 결과를 보고 실제로 어떤 행동을 바꾸는지 본다.

- 진단 후 비교 페이지 이동 여부
- 대체 후보 확인 여부
- 저장 기능 사용 여부

## 남은 보완

- 행정동/상권 수요 데이터의 최신화
- 매물 단위 입력에서 업종 가정 반영
- 저장된 검토 결과의 공유/PDF 출력

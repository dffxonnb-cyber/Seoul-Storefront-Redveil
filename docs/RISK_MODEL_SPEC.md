# 리스크 모델 정의

## 목표

이 모델은 `어디를 사야 하는가`를 답하는 모델이 아닙니다.

대신 아래 질문에 답합니다.

> 이 매입 판단이 틀릴 수 있는 가장 강한 이유는 무엇인가?

## 모델링 원칙

- 설명 가능한 점수를 사용하고, 불투명한 예측처럼 보이지 않게 합니다.
- 총점만 보여주지 않고 근거를 함께 남깁니다.
- 과한 정밀함보다 강건한 휴리스틱을 우선합니다.
- 결과는 최종 추천이 아니라 매입 전 경고 신호로 사용합니다.

## 1단계 점수 구조

### 총 리스크 점수

- 0~100 범위 점수
- 높을수록 매입 보류 근거가 강함

### 축별 점수

- 가격 부담 리스크
- 거래 유동성 리스크
- 가격 변동성 리스크
- 상권 과밀 리스크

## 보류 사유 생성 방식

- 축별 점수 중 상위 항목을 우선 노출합니다.
- 총점과 함께 문장형 보류 사유를 생성합니다.
- 같은 점수라도 표본이 얇으면 신뢰도 경고를 추가합니다.

## 출력 결과

- 총 리스크 점수
- 리스크 유형
- 문장형 보류 사유
- 대체 후보
- 현장 확인 체크리스트

## 주의점

- 개별 건물 임대차 조건과 관리 상태는 포함하지 않습니다.
- 수요 데이터와 거래 데이터의 시점이 완전히 같지는 않습니다.
- 현재는 서울 구 단위 판단에 집중합니다.

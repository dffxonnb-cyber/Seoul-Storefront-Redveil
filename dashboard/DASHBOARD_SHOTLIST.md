# Dashboard Shot List

## 1. Overview

- 목적: 프로젝트가 `추천`이 아니라 `리스크 판별` 서비스라는 점을 한 장에서 설명
- 포함 요소:
- `Immediate risk queue`
- `Current case studies`
- 시점 불일치 경고

## 2. District Risk Review

- 목적: 한 구를 선택했을 때 `총점 -> 반대 메모 -> 근거 -> 대체 후보` 흐름이 보이게 저장
- 추천 구:
- `서초구`
- 이유: 현재 최고 위험 구이며 objection이 가장 분명함

## 3. Case Study

- 목적: 포트폴리오용 서술형 분석 화면 확보
- 추천 구:
- `강남구` 또는 `중구`
- 포함 요소:
- 6개월 가격 변화
- 6개월 거래 변화
- 대체 후보
- 현장 확인 체크리스트

## 4. Demand Fragility

- 목적: 수요/매출 기반 분석 축을 별도로 보여주기
- 포함 요소:
- `Very High`, `High` 필터 결과
- 상권명, 수요 취약 점수, objection

## 5. Admin-Dong Saturation

- 목적: 구 단위 점수 뒤에 실제 행정동 밀집도가 있다는 점을 보여주기
- 추천 구:
- `강남구` 또는 `마포구`
- 포함 요소:
- `stores per admin dong`
- `food-store share`
- 상위 행정동 목록

## 6. Optional

- `CASE_STUDIES.md` 일부를 캡처해서 문서형 산출물도 같이 제시
- `KPI_DEFINITION.md`의 현재 스냅샷을 캡처해서 프로젝트 완성도를 보여주기

# User Journey

## 핵심 사용자

- 서울 소형 상가를 검토하는 개인 투자자
- 중개사 설명은 들었지만 숫자로 다시 검증하고 싶은 사용자
- 현장 답사 전에 빠른 1차 필터가 필요한 사용자

## 사용자 여정

### 1. 유입

사용자는 `좋아 보이는 구` 또는 `검토 중인 매물`을 이미 하나 들고 들어온다.
문제는 정보 부족이 아니라, 좋은 이야기만 먼저 들었다는 점이다.

### 2. 1차 필터

사용자는 3분 진단이나 내 매물 검토에서 다음을 입력한다.

- 구
- 희망 매입가
- 면적
- 보유 기간
- 우선순위

출력:

- 매입 보류 여부
- 리스크 유형
- 핵심 반대 근거

### 3. 현장 검증 준비

사용자는 서비스가 제시한 체크리스트를 본다.

- 가격선 재확인
- 거래 회전 속도 점검
- 상권 과밀 확인
- 공실/임차인 교체 확인

### 4. 대체 검토

사용자는 같은 예산에서 더 안전한 대체 구를 비교한다.
이 단계에서 서비스는 단순 경고 도구가 아니라 `비교 도구`가 된다.

### 5. 저장 및 회고

사용자는 검토 결과를 저장한다.
이 저장 기능이 있어야:

- 여러 매물을 이어서 비교할 수 있고
- 나중에 어떤 판단을 했는지 다시 볼 수 있으며
- 실제 서비스처럼 반복 사용이 가능하다

## 실패 경험을 줄이는 포인트

- 추천보다 반대 근거를 먼저 보여준다
- 점수보다 문장형 메모를 먼저 보여준다
- 경고 다음에 바로 대체 후보를 붙인다
- 현장 체크리스트를 서비스 안에서 바로 연결한다

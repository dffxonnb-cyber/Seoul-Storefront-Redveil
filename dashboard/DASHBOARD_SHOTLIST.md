# 대시보드 캡처 목록

## 1. 전체 개요

- 목적: 이 프로젝트가 `추천`이 아니라 `리스크 판별` 서비스라는 점을 한 장에서 설명
- 포함 요소:
- `즉시 확인할 리스크 큐`
- `현재 케이스 스터디`
- 시점 불일치 경고

## 2. 구 리스크 리뷰

- 목적: 한 구를 선택했을 때 `총점 -> 보류 메모 -> 근거 -> 대체 후보` 흐름이 보이게 저장
- 추천 구:
- `서초구`
- 이유: 현재 최고 위험 구이며 보류 사유가 가장 분명함

## 3. 케이스 스터디

- 목적: 서술형 분석 화면을 확보
- 추천 구:
- `강남구` 또는 `중구`
- 포함 요소:
- 6개월 가격 변화
- 6개월 거래 변화
- 대체 후보
- 현장 확인 체크리스트

## 4. 수요 취약 화면

- 목적: 수요 기반 분석 축을 별도 화면으로 보여주기
- 포함 요소:
- `매우 높음`, `높음` 필터 결과
- 상권명, 수요 취약 점수, 보류 사유

## 5. 행정동 과밀 화면

- 목적: 구 단위 점수 아래에서 실제 행정동 밀집도가 다르다는 점을 보여주기
- 추천 구:
- `강남구` 또는 `마포구`
- 포함 요소:
- `행정동당 점포 수`
- `음식점 비중`
- 상위 행정동 목록

## 6. 선택 제출 자료

- `CASE_STUDIES.md` 화면을 캡처해서 문서형 결과물로 함께 제시
- `KPI_DEFINITION.md`를 캡처해서 프로젝트 완성도를 보조 설명
